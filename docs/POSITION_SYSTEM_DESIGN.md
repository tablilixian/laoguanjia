## 一、需求背景

### 1.1 问题描述

现有位置系统存在以下问题：
- 位置描述不够直观，难以快速定位物品
- 缺少可视化辅助，无法"看到"物品在哪里
- 层级结构不够灵活，不支持精细化定位

### 1.2 用户需求

用户的核心诉求：
1. **方便增删位置**：能快速添加、删除位置
2. **直观的位置描述**：知道物品在家里哪个地方
3. **示意图辅助**：通过可视化方式理解位置
4. **灵活的定位方式**：
   - 开放空间（客厅、卧室）→ 东南西北 + 上下
   - 封闭容器（书架、抽屉）→ 编号、格子

### 1.3 设计约束

- 位置层级：最多 6 层（家 → 房间 → 家具 → 格子 → 抽屉 → 物品）
- 不要求精确坐标，模糊描述即可
- 可视化主要用于查找物品，不在创建物品时强制填写
- 示意图交互：点击选择 + 文字输入 双模式

## 二、核心设计理念

### 2.1 核心概念

**"位置 = 容器 + 定位方式"**

每个位置具有双重身份：
- **作为容器**：可以包含子位置和物品
- **作为被定位的点**：在父容器中有自己的位置

### 2.2 两种定位模式

| 模式 | 适用场景 | 定位方式 | 示例 |
|------|----------|----------|------|
| 方向型 | 开放空间 | 东南西北 + 上下 | 客厅的东南角上层 |
| 编号型 | 封闭容器 | 第1层、第2格 | 书架的第3层 |

### 2.3 设计原则

1. **渐进增强**：基础功能简洁，高级功能可选
2. **智能建议**：根据父位置自动推荐合适的模板
3. **所见即所得**：示意图与文字描述实时同步
4. **灵活可调**：预设模板可调整参数

## 三、模板系统

### 3.1 模板类型

| 类型 | 英文标识 | 适用场景 | 定位方式 |
|------|----------|----------|----------|
| 方向型 | `direction` | 客厅、卧室、书房 | 东南西北 + 上中下 |
| 编号型 | `index` | 书架、抽屉、柜子 | 第1层、第2格 |
| 网格型 | `grid` | 收纳盒、抽屉内部 | A1、B2、C3 |
| 堆叠型 | `stack` | 盒子、箱子 | 上层、中层、下层 |
| 无模板 | `none` | 普通位置 | 纯文字描述 |

### 3.2 方向型模板变体

#### direction_4（四向，无高度）
```
    北
    ↑
西 ← → 东
    ↓
    南
```
- 位置数：4 或 5（含中心）

#### direction_4h（四向 + 上下）
- 位置数：4×3 = 12 或 5×3 = 15

#### direction_8（八向，无高度）
```
    西北  北  东北
      ↖   ↑   ↗
  西  ← ──┼── →  东
      ↙   ↓   ↘
    西南  南  东南
```
- 位置数：8 或 9（含中心）

#### direction_8h（八向 + 上下）
- 位置数：8×3 = 24 或 9×3 = 27

### 3.3 编号型模板变体

#### index_shelf（书架）
```
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ 第4层│ │ 第4层│ │ 第4层│ │ 第4层│
├─────┤ ├─────┤ ├─────┤ ├─────┤
│ 第3层│ │ 第3层│ │ 第3层│ │ 第3层│
├─────┤ ├─────┤ ├─────┤ ├─────┤
│ 第2层│ │ 第2层│ │ 第2层│ │ 第2层│
├─────┤ ├─────┤ ├─────┤ ├─────┤
│ 第1层│ │ 第1层│ │ 第1层│ │ 第1层│
└─────┘ └─────┘ └─────┘ └─────┘
```
- 配置参数：`totalSlots`（总层数）、`columns`（每行几个）

#### index_drawer（抽屉）
```
┌─────────┐
│ 抽屉 3  │
├─────────┤
│ 抽屉 2  │
├─────────┤
│ 抽屉 1  │
└─────────┘
```
- 配置参数：`totalSlots`（抽屉数）、`namingPattern`（命名模板）

### 3.4 网格型模板变体

#### grid_3x3（3×3格子）
```
    A    B    C
┌────┬────┬────┐
│ A1 │ B1 │ C1 │  ← 第1行
├────┼────┼────┤
│ A2 │ B2 │ C2 │  ← 第2行
├────┼────┼────┤
│ A3 │ B3 │ C3 │  ← 第3行
└────┴────┴────┘
```
- 配置参数：`rows`（行数）、`cols`（列数）、`rowLabels`、`colLabels`

#### grid_locker（更衣柜/鞋柜）
```
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│  1  │ │  2  │ │  3  │ │  4  │
└─────┘ └─────┘ └─────┘ └─────┘
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│  5  │ │  6  │ │  7  │ │  8  │
└─────┘ └─────┘ └─────┘ └─────┘
```
- 配置参数：`rows`、`cols`、`sequentialNumbering`（顺序编号）

### 3.5 堆叠型模板

#### stack_3（三层堆叠）
```
┌─────────────┐
│    上层     │
├─────────────┤
│    中层     │
├─────────────┤
│    下层     │
└─────────────┘
```
- 配置参数：`levels`（层数）、`labels`（层标签）

## 五、子位置模板建议机制

### 5.1 建议规则

创建子位置时，根据父位置名称智能推荐模板：

| 父位置关键词 | 建议模板 | 备选模板 |
|--------------|----------|----------|
| 衣柜、橱柜、浴室柜 | index（3层抽屉） | index（2层/4层） |
| 书架、书柜 | index（4-6层） | index（3层） |
| 斗柜 | index（4层） | index（2-6层） |
| 鞋柜 | index（4格） | index（2-8格） |
| 收纳盒、收纳箱 | grid（3×3） | stack（3层） |
| 抽屉内部 | grid（2×3） | grid（自定义） |
| 抽屉 | index（层数可调） | - |

### 5.2 建议交互

```
创建子位置：衣柜
              │
              ▼
┌─────────────────────────────┐
│  系统建议                    │
│                             │
│  📊 建议使用"编号型"模板      │
│  ┌─────────────────────┐   │
│  │  3层抽屉            │   │
│  │  ├─ 第3层           │   │
│  │  ├─ 第2层           │   │
│  │  └─ 第1层           │   │
│  └─────────────────────┘   │
│                             │
│  [接受建议] [换一个] [自定义] │
└─────────────────────────────┘
```

### 5.3 用户可调整参数

接受建议后，用户可调整：
- 层数：2层、3层、4层、5层、6层
- 命名方式：第{n}层 / {n}F / L{n}

## 七、数据模型

### 7.1 位置表扩展

**表名：** `item_locations`

**新增字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `template_type` | TEXT | 模板类型：direction/index/grid/stack/none |
| `template_config` | JSONB | 模板配置参数 |
| `position_in_parent` | JSONB | 在父级中的位置 |
| `position_description` | TEXT | 位置描述（冗余存储，用于快速显示） |

### 7.2 模板配置结构

**方向型配置：**
```json
{
  "template": "direction",
  "directions": {
    "enabled": true,
    "labels": ["东", "南", "西", "北", "东北", "西北", "东南", "西南"],
    "includeCenter": true
  },
  "heights": {
    "enabled": true,
    "labels": ["上层", "中层", "下层"]
  }
}
```

**编号型配置：**
```json
{
  "template": "index",
  "totalSlots": 4,
  "startFrom": 1,
  "namingPattern": "第{n}层",
  "columns": 4
}
```

**网格型配置：**
```json
{
  "template": "grid",
  "rows": 3,
  "cols": 3,
  "rowLabels": ["1", "2", "3"],
  "colLabels": ["A", "B", "C"]
}
```

**堆叠型配置：**
```json
{
  "template": "stack",
  "levels": 3,
  "labels": ["上层", "中层", "下层"]
}
```

### 7.3 位置在父级中的定位

```json
{
  "direction": "east",
  "height": "top"
}
```
或
```json
{
  "index": 2

```

## 十、子位置创建界面设计

### 10.1 设计目标

创建子位置时，用户应该能够：
1. **直观看到父位置信息**：知道正在为哪个位置添加子位置
2. **选择新位置在父级中的位置**：通过示意图选择槽位
3. **预览位置描述**：实时看到完整的位置描述
4. **理解层级关系**：通过可视化方式理解位置层级

### 10.2 理想界面布局


### 10.3 核心交互流程

```
1. 用户点击"添加子位置"
   │
   ▼
2. 系统获取父位置信息（名称、图标、模板类型）
   │
   ▼
3. 界面显示父位置卡片 + 父位置示意图
   │
   ▼
4. 用户输入子位置名称
   │
   ├── 名称匹配模板建议 → 自动选择定位方式
   ▼
5. 用户选择"在父级中的位置"
   │
   ├── 点击父位置示意图的槽位
   │      或
   └── 直接选择方向/高度选项
   │
   ▼
6. 实时预览位置描述
   │
   ▼
7. 用户选择子位置自己的定位方式（可选）
   │
   ▼
8. 保存时自动计算：
   - position_in_parent: 新位置在父级中的槽位
   - position_description: 完整位置描述
   - path: 完整路径 "主卧/东北角/中层/衣柜"
```

### 10.4 父级信息显示组件

**组件名称：** `ParentLocationCard`

**功能：**
- 显示父位置名称、图标
- 渲染父位置的示意图（只读模式）
- 显示父位置已有的子位置（如果有）

**显示逻辑：**
```dart
// 伪代码
Widget build(BuildContext context) {
```

### 10.5 槽位选择组件

**组件名称：** `ParentSlotSelector`

**功能：**
- 根据父位置模板类型显示对应的选择器
- 支持方向型、编号型、网格型、堆叠型
- 实时显示选中位置
- 支持清空选择

**位置选择类型：**

| 父位置模板 | 选择器类型 | 选择方式 |
|------------|------------|----------|
| direction | DirectionPicker | 点击方向 + 高度 Tab |
| numbering | IndexPicker | 点击编号列表 |
| grid | GridPicker | 点击二维网格 |
| stack | StackPicker | 点击层级列表 |

### 10.6 位置描述预览

**组件名称：** `PositionDescriptionPreview`

**功能：**
- 实时拼接位置描述
- 格式：`{父位置}的{槽位描述}{位置名称}`
- 示例：`主卧的东北角中层衣柜`

**描述生成逻辑：**
```dart
String generateDescription({
  required ItemLocation parent,
  required Map<String, dynamic> slotInParent,
  required String locationName,
}) {
  final parts = <String>[];
  
  // 父位置名称
  parts.add(parent.name);
  
  // 槽位描述
  final slotDesc = generateSlotDescription(slotInParent, parent.templateType);
  if (slotDesc.isNotEmpty) {
    parts.add(slotDesc);
  }
  
  // 新位置名称
  parts.add(locationName);
  
  return parts.join('的');
}
```

### 10.7 数据保存逻辑

**保存时自动计算：**

```dart
Future<void> saveLocation(LocationFormData data) async {
  // 1. 如果有父位置和槽位选择
  if (data.parentId != null && data.slotInParent != null) {
    // 计算 position_in_parent
    data.positionInParent = data.slotInParent;
    
    // 计算 position_description
    final parent = await getLocation(data.parentId);
    data.positionDescription = generateDescription(
      parent: parent,
      slotInParent: data.slotInParent,
      locationName: data.name,
    );
    
    // 计算 path
    final parentPath = parent.path ?? parent.id;
    data.path = '$parentPath/${generatePathSegment(data.slotInParent)}';
  }
  
  // 2. 保存位置
  await saveToDatabase(data);
}
```

### 10.8 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 父位置无模板 | 不显示槽位选择器 |
| 父位置槽位已满 | 提示"该槽位已被占用" |
| 用户跳过槽位选择 | 允许，但 position_in_parent 为 null |
| 编辑已有位置 | 加载已有的 position_in_parent，显示选中状态 |
