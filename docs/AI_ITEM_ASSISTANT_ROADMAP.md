## 1. 愿景与目标

**愿景**：打造一个智能化的家庭物品管理助手，让用户通过自然对话就能完成物品的录入、查询、统计和管理。

**核心价值**：
- 🤖 **智能化**：AI 驱动的意图识别和自然语言处理
- ⚡ **高效**：一句话完成复杂操作
- 🎯 **准确**：模糊匹配，容错率高
- 💡 **洞察**：智能分析，发现潜在价值

## 3. P0 - 紧急/重要

### 3.1 意图识别优化

**问题**：当前关键词匹配方式不够准确，容易误判

**改进方案**：
- [ ] 使用 AI 模型进行意图识别
- [ ] 支持多意图识别（一句多意）
- [ ] 添加置信度判断，不确定时询问用户

**示例**：
```
用户：帮我把冰箱保存到客厅，顺便看看客厅有哪些东西

当前：只能识别一个意图
目标：同时识别「创建物品」和「查询物品」两个意图
```

### 3.2 位置层级支持

**问题**：只支持精确位置匹配，父级位置查询不准确

**改进方案**：
- [ ] 支持「客厅有哪些物品」→ 包含客厅及其子位置的所有物品
- [ ] 支持路径匹配（如「厨房 > 橱柜 > 第一层」）
- [ ] 添加位置面包屑显示

### 3.3 创建结果预览

**问题**：创建后直接保存，用户无法确认

**改进方案**：
- [ ] 创建前显示预览卡片
- [ ] 用户确认后再保存
- [ ] 支持修改后保存

**UI 设计**：
```
┌─────────────────────────────────┐
│  📦 即将创建：冰箱               │
├─────────────────────────────────┤
│  类型: 🔌 家电                   │
│  位置: 🏠 客厅                  │
│  品牌: [输入框]                  │
│  型号: [输入框]                  │
├─────────────────────────────────┤
│  [取消]           [确认创建]     │
└─────────────────────────────────┘
```

## 4. P1 - 重要

### 4.1 批量标签操作

**功能**：
- [ ] 「给客厅所有物品打上'常用'标签」
- [ ] 「移除所有'借出'标签」
- [ ] 「把有'需要维修'标签的物品移到储藏室」

### 4.2 位置创建

**功能**：
- [ ] 「在厨房添加一个新位置叫'调料柜'」
- [ ] 支持创建时同时指定父位置
- [ ] 创建后自动使用新位置保存物品

### 4.3 AI 引导式对话

**功能**：
- [ ] 创建物品时，缺少信息时主动询问
- [ ] 「您想把空调保存到哪里？」→ 等待用户输入位置
- [ ] 对话上下文保持

**示例**：
```
用户：帮我保存空调
AI：请告诉我保存位置，例如「客厅」或「主卧」
用户：客厅
AI：好的，已将「空调」保存到「客厅」
```

### 4.4 语音输入支持

**功能**：
- [ ] 集成语音识别（ASR）
- [ ] 支持语音输入替代打字
- [ ] 语音播报 AI 响应（TTS）

### 4.5 物品图片识别

**功能**：
- [ ] 拍照识别物品（拍照 → AI 识别 → 自动填充信息）
- [ ] 支持从相册选择图片
- [ ] 批量拍照识别

## 5. P2 - 一般

### 5.1 断舍离助手

**功能**：
- [ ] 「找出超过一年没用过的物品」
- [ ] 「建议可以捐赠的物品」
- [ ] 「哪些物品可以卖掉」

**分析维度**：
- 最后使用时间（需记录使用日志）
- 物品状态
- 用户标记的「处理意向」标签

### 5.2 智能提醒

**功能**：
- [ ] 「哪些物品保修快到期了？」→ 直接显示提醒卡片
- [ ] 「需要补充的消耗品」→ 基于消耗频率预测
- [ ] 「换季提醒」→ 自动推荐换装物品

### 5.3 物品比较

**功能**：
- [ ] 「比较一下厨房的两台空调」
- [ ] 展示参数对比表格
- [ ] 支持同类型物品横向比较

### 5.4 自然语言筛选

**功能**：
- [ ] 「显示所有价格超过1000的家电」
- [ ] 「找出购买日期在2025年的物品」
- [ ] 「筛选保修期内的物品」

### 5.5 对话历史

**功能**：
- [ ] 保存历史对话记录
- [ ] 支持查看和继续历史对话
- [ ] 导出对话记录
